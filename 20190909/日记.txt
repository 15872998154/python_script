15.大获成功,原来不能仅仅通过aitomator viewer来进行定位，通过这个工具定位定位到的元素，
	按照在appium session中的serch for element中未必能够查找到，所以还是得要严格的
	按照appium session中的标准来进行元素定位
	比如在小木虫这款app中，登录页面整个是无法选取定位的
	解决方案则是通过坐标来点击屏幕的指定位置,由于不能设备屏幕大小不同，而代码考虑兼容，则有方法如下：
	假设想要点击的坐标为 (300,200)     (横向,纵向)
	width = driver.get_window_size()['width'] 
	height = driver.get_window_size()['height']	
	相对系数 a1 = 300 / width 
		   	b1 = 200 / height
	最终点击的位置为
	x = a1 * driver.get_window_size()['width']
	y = b1 * driver.get_window_size()['height']

26.真机与模拟器尽量保持分辨率相同,夜神模拟器支持自定义屏幕分辨率，分辨率不同，即使按照比例来进行坐标的点击,但是分辨率本身就不是宽高乘以一个系数得来的。
而且在真机测试的时候，真机是1080*2160的分辨率，而模拟器分辨率760的，所以导致画面不是成相同比例的，在模拟器上，登录按钮显然靠屏幕上方一下。所以即使按照代码，等比例的进行点击，依然无法点击到我们想要点击的地方。
